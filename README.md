# CLECMAS

CLECMAS 2.0 is a governed AI-assisted compliance operations platform for continuing education and adjacent regulated training evidence.

## Current Repository Status

This public repository is currently initialized for a safe baseline scaffold only.

Current limits:

- First production scope is under 50 professionals.
- Repository creation does not authorize build start.
- Repository creation does not authorize deployment readiness.
- Repository creation does not authorize production readiness.
- Repository creation does not authorize AI-governance readiness.
- Repository creation does not authorize Codex execution.
- Project Intake Packet, IT Intake Packet, and AI Governance Appendix remain pending until returned and incorporated through controlled update.

## Product Boundary

CLECMAS supports:

- professional profiles
- licenses and jurisdictions
- versioned approved rules
- evidence upload and review
- deterministic compliance evaluation
- remaining-obligation calculation
- due-soon and overdue alerts from deterministic trigger data
- audit logs
- audit packets
- submission-readiness handoff
- review-gated AI-assisted rules intelligence and narrative support

CLECMAS does not:

- provide legal advice
- make final regulatory determinations
- publish rules without human approval
- approve evidence without human review
- change compliance status by AI output
- submit filings
- pay fees
- sign
- certify
- attest
- make penalty-of-perjury declarations
- delete evidence without controlled disposition
- alter audit history

Licensed professionals retain responsibility for final submission, payment, signature, certification, attestation, and penalty-of-perjury declarations.

## Runtime Resilience

Approved rule versions, reviewed evidence, deterministic status calculation, alerts, audit packets, and submission-readiness views remain usable if the AI provider is temporarily unavailable.

## Repository Hygiene

Never commit:

- client data
- secrets
- uploaded evidence
- local database files
- vector artifacts
- generated caches
- exported packets containing live data
